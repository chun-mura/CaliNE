# コミット

現在の変更を意味のある論理単位に分割してコミットしてください。

## 手順

1. `git status` と `git diff` で変更内容を確認する
2. `git log --oneline -5` で直近のコミットメッセージのスタイルを確認する
3. 変更を意味のある単位に分類する（例: インフラ / アプリケーションコード / ドキュメント）
4. 分類ごとにステージング → コミットを実行する

## コミットメッセージのルール

- 英語で記述する
- 1行目は動詞で始める（add, update, fix, remove, refactor 等）
- 1行目は簡潔に（70文字以内）
- 必要に応じて空行の後に日本語で補足説明を追加する
- 末尾に `Co-Authored-By: Claude Opus 4.6 (1M context) <noreply@anthropic.com>` を付与する

## 注意事項

- `.env`、シークレット、`*.tfstate` 等の機密ファイルをコミットしない
- 変更がない場合は空コミットを作成しない
- `git push` は実行しない（ユーザーが明示的に指示した場合のみ）
- コミット後に `git status` で結果を確認する
